01 · The challenge

From idea to demo in 90 minutes

Your team’s mission is simple: identify a real problem, build a working prototype with AI, and share a live link.

01

Choose a problem

Pick something related to teaching, research, faculty productivity, student life or the future of computing.

02

Build with AI

Use any AI tool. You do not need to write all the code yourself. The point is to collaborate effectively with AI.

03

Publish a prototype

Your output should be demonstrable. A webpage, interactive tool, dashboard, visualiser or working simulation all count.

04

Share and vote

Add your link to the shared submission page. Everyone explores the projects and votes for their favourite.

Definition of success

It works. It is useful. You can demo it.

Do not optimise for production quality. Optimise for a clear idea, a compelling interaction and a memorable demo.

04 · Starter prompts

Copy, adapt and build

Strong prompts reduce setup time. Start with a clear audience, a narrow job and a visible interaction.

Universal web app prompt 
Build a polished, responsive web application for [AUDIENCE] that helps them [JOB TO BE DONE].

The app should:
- have one clear primary interaction
- use realistic sample data
- show the result visually
- include helpful empty, loading and success states
- feel like a real product, not a wireframe
- work without a backend where possible

Use a modern, minimal interface. Keep the first version small enough to finish quickly.
Improve an existing prototype 
Review this prototype as a product designer and senior engineer.

Improve:
1. visual hierarchy
2. clarity of the main user journey
3. demo impact
4. mobile responsiveness
5. error handling
6. realism of sample data

Do not add unnecessary features. Prioritise changes that make the prototype easier to understand in a 60-second demo.
Fix and deploy 
Check the project for errors, run it locally, fix any broken functionality, and prepare it for deployment.

Then:
- add a concise README
- verify all buttons work
- remove placeholder text
- ensure the homepage explains the value in under 10 seconds
- provide the simplest deployment steps for a public link
